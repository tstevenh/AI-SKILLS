# Launch Checklists

Ready-to-use checklists for different launch types. Copy, customize, and execute.

---

## Major Product Launch Checklist

### 2 Weeks Before Launch

**Product Readiness:**

- [ ] Feature complete and documented
- [ ] All bugs from beta testing fixed
- [ ] User onboarding flow tested
- [ ] Demo/tutorial content created
- [ ] FAQ section updated
- [ ] Support documentation complete

**Marketing Preparation:**

- [ ] Landing page updated with new features
- [ ] Comparison content created (what's new)
- [ ] Social media content calendar drafted
- [ ] Email announcement sequence written
- [ ] Press kit updated (if applicable)
- [ ] Screenshots/videos captured

**Technical Setup:**

- [ ] Staging environment tested
- [ ] Rollback plan documented
- [ ] Analytics tracking verified
- [ ] Payment/checkout flow tested
- [ ] Email delivery tested

### 1 Week Before Launch

**Beta & Validation:**

- [ ] Beta testers provided final feedback
- [ ] Critical issues addressed
- [ ] Testimonials collected from beta users
- [ ] Case study drafted (if applicable)

**Marketing Finalization:**

- [ ] All social posts scheduled
- [ ] Email sequences loaded and scheduled
- [ ] Landing page final review complete
- [ ] Pricing finalized and updated
- [ ] Community announcement drafted

**Team Coordination:**

- [ ] Support team briefed on new features
- [ ] Community moderators notified
- [ ] Launch day schedule confirmed
- [ ] Emergency contacts listed

### Launch Day

**Morning (Before Announcement):**

- [ ] Final production deployment
- [ ] Smoke test all critical paths
- [ ] Verify payment processing works
- [ ] Confirm email delivery working
- [ ] Landing page live and correct

**Launch Sequence:**

- [ ] Push product update live
- [ ] Update landing page
- [ ] Publish changelog/release notes
- [ ] Send email to existing customers
- [ ] Post main announcement on Twitter/X
- [ ] Share in community channels
- [ ] Notify beta testers it's live

**Post-Launch (Same Day):**

- [ ] Monitor for critical issues
- [ ] Respond to all social comments
- [ ] Answer community questions
- [ ] Track initial sales/signups
- [ ] Thank early adopters publicly

### Week After Launch

**Day 1-2:**

- [ ] Review overnight feedback
- [ ] Fix any reported issues
- [ ] Continue social engagement
- [ ] Monitor support tickets

**Day 3-4:**

- [ ] Send follow-up email with tips
- [ ] Publish tutorial content
- [ ] Collect new testimonials
- [ ] Review initial metrics

**Day 5-7:**

- [ ] Complete launch metrics review
- [ ] Document learnings
- [ ] Plan iteration based on feedback
- [ ] Schedule post-launch content

## Email Launch Sequence Checklist

### Pre-Launch Emails

**Teaser (3-5 days before):**

- [ ] Subject line creates curiosity
- [ ] Body hints without revealing
- [ ] Date/time of launch mentioned
- [ ] No CTA needed yet

**Reminder (1 day before, optional):**

- [ ] Subject line builds anticipation
- [ ] Brief preview of value
- [ ] Exact launch time mentioned

### Launch Day Emails

**Main Announcement:**

- [ ] Clear, benefit-focused subject
- [ ] What's new (headlines)
- [ ] How to access/upgrade
- [ ] Social proof (if available)
- [ ] Clear CTA button
- [ ] P.S. with urgency or bonus

**Existing Customer Email (if different):**

- [ ] Acknowledge their status
- [ ] Explain what they get
- [ ] Show how to access
- [ ] Reinforce value of being early

### Post-Launch Emails

**Day 2-3 Tips Email:**

- [ ] Subject: Getting started tip
- [ ] Quick win they can achieve
- [ ] Link to full documentation
- [ ] Support option mentioned

**Day 5-7 Follow-up:**

- [ ] Subject: Results/testimonials
- [ ] Early wins from customers
- [ ] Feature highlights
- [ ] For non-buyers: gentle nudge

## Product Hunt Launch Checklist

### 2 Weeks Before

**Product Hunt Preparation:**

- [ ] Hunter identified (or self-hunting)
- [ ] Product page drafted
- [ ] Tagline finalized (60 chars max)
- [ ] Description written (260 chars)
- [ ] Gallery images created (1270x760px)
- [ ] First comment drafted
- [ ] Maker's story prepared

**Community Prep:**

- [ ] Supporters notified of launch date
- [ ] Community primed to engage
- [ ] Time zones of supporters considered

### 1 Week Before

- [ ] Product page reviewed and finalized
- [ ] All assets uploaded
- [ ] Launch day schedule created
- [ ] Support availability confirmed
- [ ] Backup plans for issues

### Launch Day (12:01 AM PT)

**Hour 1:**

- [ ] Product goes live
- [ ] First comment posted
- [ ] Share on Twitter
- [ ] Notify email list
- [ ] Post in community

**Throughout Day:**

- [ ] Respond to all PH comments
- [ ] Update social with progress
- [ ] Thank supporters
- [ ] Share user feedback
- [ ] Monitor for issues

**End of Day:**

- [ ] Thank community regardless of result
- [ ] Screenshot final position
- [ ] Plan follow-up content

### Post-Launch

- [ ] Send thank you email to supporters
- [ ] Write retrospective
- [ ] Update landing page with badge (if applicable)
- [ ] Continue engaging new users

## Pre-Launch Validation Checklist

Before any launch, verify:

### Technical Verification

- [ ] Product works as expected
- [ ] No critical bugs
- [ ] Checkout/payment flow works
- [ ] Email delivery working
- [ ] Analytics tracking active
- [ ] Support system ready

### Content Verification

- [ ] All copy proofread
- [ ] Links working
- [ ] Images loading
- [ ] Mobile responsive
- [ ] SEO elements present
- [ ] Legal pages updated

### Process Verification

- [ ] Team knows their roles
- [ ] Communication channels ready
- [ ] Emergency contacts available
- [ ] Rollback plan exists
- [ ] Success metrics defined

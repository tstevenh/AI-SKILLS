# Design QA

Comprehensive visual quality assurance for ensuring design consistency and accuracy.

## What It Does

This skill provides methodologies for design quality assurance—visual regression testing, pixel-perfect comparison, responsive design testing, and maintaining design system consistency across platforms.

## Features

- Visual regression testing frameworks
- Pixel-perfect comparison techniques
- Responsive design testing across devices
- Typography and color consistency checks
- Component consistency validation
- Dark mode and accessibility visual checks
- Cross-browser rendering validation
- Figma-to-code comparison methods
- Design token validation

## Installation

1. Copy the `design-qa/` folder into your OpenClaw skills directory
2. Restart OpenClaw

## Usage

Use this skill when:
- Testing visual consistency across pages
- Performing pixel-perfect design reviews
- Validating responsive behavior
- Checking cross-browser compatibility
- Auditing design system implementation
- Comparing Figma designs to live code

## Requirements

- Visual regression testing tool (Percy, Chromatic, Applitools)
- Browser testing tools (BrowserStack, Sauce Labs)
- Design system documentation
- Access to design files (Figma, Sketch, Adobe XD)
- Screenshot comparison tools

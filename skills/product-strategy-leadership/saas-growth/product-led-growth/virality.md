# Product Virality

## Introduction

Product virality is the holy grail of PLG—when your product spreads itself through usage. Unlike paid marketing where growth costs money, viral growth compounds: every user brings more users, who bring more users. This isn't about making things "go viral" on social media—it's about building systematic, repeatable viral loops into your product.

This guide covers viral loops, invite mechanics, collaboration features, powered-by badges, share/embed features, and how to calculate and optimize your viral coefficient.

---

## 1. Viral Loops in SaaS

### What is a Viral Loop?

A viral loop is a mechanism where using your product naturally exposes new potential users to it, who then become users themselves, continuing the cycle.

```
User A uses product
        ↓
Product exposes to User B (through usage)
        ↓
User B sees value/curiosity
        ↓
User B signs up
        ↓
User B uses product
        ↓
Product exposes to User C...
        
[Cycle repeats]
```

### Types of Viral Loops

**1. Inherent Virality**

The product REQUIRES sharing to function:
- Calendly: Must send link to schedule
- Zoom: Must send link to meet
- Slack: Must invite to collaborate
- DocuSign: Must send to get signature

**Characteristics:**
- Strongest viral loop
- Built into core functionality
- Non-optional exposure
- High-quality leads (needed the function)

**2. Collaboration Virality**

Product is better with others:
- Notion: Shared workspaces
- Figma: Real-time collaboration
- Google Docs: Multi-editor docs
- Trello: Team boards

**Characteristics:**
- Value increases with team
- Invites are value-add, not spam
- Natural team expansion
- Network effects compound

**3. Word-of-Mouth Virality**

Users tell others because it's good:
- "You should try [Product], it's amazing"
- Organic recommendations
- Social sharing
- Reviews and testimonials

**Characteristics:**
- Slower but high trust
- Requires excellent product
- No forced mechanics
- Best long-term referral source

**4. Incentivized Virality**

Users are rewarded for sharing:
- Dropbox: Free storage for referrals
- Uber: Free rides for referrals
- Referral programs with credits

**Characteristics:**
- Can accelerate growth
- Risk of low-quality referrals
- Works best layered on good product
- Incentive must be compelling

**5. Artificial/Exposure Virality**

Product creates external visibility:
- "Powered by Calendly" in emails
- Mailchimp badge in email footers
- "Made with Webflow" on sites
- Watermarks on free tier

**Characteristics:**
- Passive awareness
- Lower conversion but high volume
- Brand building
- Free advertising

### Viral Loop Examples

**Calendly Viral Loop:**
```
1. User creates scheduling link
2. User sends link to contacts
3. Contacts see Calendly (brand exposure)
4. Contacts book meeting
5. Contacts think "I need this too"
6. Some contacts sign up
7. New users create scheduling links
8. [Loop continues]
```

**Slack Viral Loop:**
```
1. Team starts using Slack
2. Team invites external collaborators
3. Collaborators join shared channel
4. Collaborators experience Slack
5. Collaborators want Slack for their team
6. New teams start using Slack
7. [Loop continues]
```

**Notion Viral Loop:**
```
1. User creates public page/template
2. Page is shared (Twitter, Reddit, etc.)
3. Viewers see Notion's capability
4. Viewers sign up to try
5. New users create and share
6. [Loop continues]
```

## 2. Invite Mechanics

### The Invite Flow

**Step 1: Prompt to Invite**

When and where to ask:
- After first success (riding the high)
- When collaboration would help
- When hitting invite-gated feature
- During onboarding

**Step 2: Easy Invite Method**

Multiple options:
- Email invite
- Shareable link
- Import contacts
- Social sharing

**Step 3: Compelling Message**

Default invite copy matters:
- Clear value proposition
- Personal feeling (from [Name])
- Easy action

**Step 4: Landing Experience**

Invited user sees:
- Why they were invited
- What they'll get
- How to start

### Invite Optimization

**Track These Metrics:**
- Invites sent per user
- Invite acceptance rate
- Invited user activation rate
- Time from invite to action

**Optimization Levers:**

| Lever | Impact | How |
|-------|--------|-----|
| Timing | High | Prompt after success |
| Ease | High | Reduce steps to invite |
| Copy | Medium | A/B test messaging |
| Incentive | Medium | Add mutual benefit |
| Reminder | Medium | Follow-up if no action |

**Invite Benchmarks:**

| Metric | Low | Typical | High |
|--------|-----|---------|------|
| % users who invite | 5% | 15-25% | 40%+ |
| Invites per inviter | 1-2 | 3-5 | 7+ |
| Invite accept rate | 10% | 20-30% | 40%+ |
| Invited activation | 30% | 50-60% | 70%+ |

## 3. Collaboration Features

### Collaboration as Viral Mechanic

Collaboration features serve dual purposes:
1. Product value: Teams work better together
2. Viral growth: Every collaborator is a potential user

### Types of Collaboration Features

**1. Shared Workspaces**

Multiple users in one environment:
- Notion workspaces
- Slack channels
- Figma projects
- Trello boards

**Viral Mechanism:**
- Inviting to workspace = new user
- Workspace expands as team uses
- Natural fit for team growth

**2. Real-Time Collaboration**

Multiple cursors, live editing:
- Google Docs
- Figma multiplayer
- Miro whiteboards
- Notion live editing

**Viral Mechanism:**
- Collaboration IS the feature
- Must invite to use feature
- Wow factor drives word-of-mouth

**3. Commenting and Feedback**

Async collaboration:
- Review requests
- Comment threads
- Approval workflows

**Viral Mechanism:**
- External reviewers exposed
- Don't need account to view (low friction)
- Account to comment (conversion point)

**4. Sharing and Publishing**

Content goes external:
- Public pages (Notion)
- Shared links (Loom)
- Embeds (Typeform)
- Published sites (Webflow)

**Viral Mechanism:**
- Shared content = brand exposure
- Viewers become curious
- Low-friction discovery

### Designing for Collaboration Virality

**Make Inviting Natural:**
- @mention invites (type name, send invite)
- Share button prominent
- "Add people" always visible
- Guest access for external

**Make Joining Easy:**
- No account required to view
- Quick account creation to interact
- SSO options
- Magic links

**Make Value Immediate:**
- Joining = instant access
- No setup required
- Context provided (why invited)
- Guided first action


## 4. Powered-by Badges

### What are Powered-by Badges?

Small brand marks that appear on user-generated output:
- "Powered by Calendly" in booking pages
- "Made with Typeform" on forms
- "Built with Webflow" on websites
- Mailchimp monkey in email footers

### Badge Effectiveness

**Why Badges Work:**
- Massive impression volume (every view = impression)
- Contextual relevance (see it when experiencing the product)
- Credibility transfer (if this good thing uses it...)
- Curiosity driver (what is this tool?)

**Badge Economics:**

```
10,000 users → Each creates content
Content viewed 100 times average = 1,000,000 impressions
0.1% click badge = 1,000 visits
5% of visitors sign up = 50 new users
Each creates content...

[Compounding growth from passive badges]
```

### Badge Strategies

**Freemium Badge (Required on Free):**
- Free users must show badge
- Paid users can remove
- Creates upgrade pressure
- Maximizes exposure

**Optional Badge:**
- Users choose to display
- Often incentivized (extra features/credits)
- Better user experience
- Lower but higher-quality exposure

**Subtle vs. Prominent:**

```
Subtle: Small "made with [Product]" text
        - Better UX
        - Less intrusive
        - May go unnoticed

Prominent: Visible banner or watermark
        - Maximum exposure
        - Can feel cheap
        - May deter paid upgrade
```

### Badge Implementation

**Where to Place:**
- Footer of output (forms, pages, embeds)
- Corner of exports (videos, images)
- Email footers (email tools)
- Share links (scheduling, documents)

**Badge Design:**
- Link to marketing page (not app)
- UTM parameters for tracking
- Mobile-friendly tap target
- Consistent brand appearance

**Example Badge HTML:**
```html
<div class="powered-by">
  <a href="https://product.com?ref=badge" target="_blank">
    <img src="badge.svg" alt="Made with Product" />
  </a>
</div>
```

### Badge Metrics

Track:
- Badge impressions (if possible)
- Badge clicks
- Signups attributed to badge (UTM)
- Conversion rate from badge traffic

## 6. Viral Coefficient Calculation

### What is Viral Coefficient (K)?

The viral coefficient (K) measures how many new users each existing user generates.

```
K = (Invites per User) × (Conversion Rate of Invites)

K = i × c

Where:
i = average invites sent per user
c = conversion rate (% of invites that become users)
```

### K Factor Examples

**Example 1: Low Virality**
```
i = 2 invites per user
c = 10% accept and sign up
K = 2 × 0.10 = 0.20

For every 100 users, you get 20 more users.
Then 20 users bring 4 more (20 × 0.2 = 4).
Then 4 bring 0.8 more...

Total from 100 original: ~125 users
Viral multiplier: 1.25x
```

**Example 2: Moderate Virality**
```
i = 4 invites per user
c = 20% accept and sign up
K = 4 × 0.20 = 0.80

100 users → 80 more → 64 more → 51 more...
Total from 100 original: ~500 users
Viral multiplier: 5x
```

**Example 3: High Virality (K > 1)**
```
i = 5 invites per user
c = 25% accept and sign up
K = 5 × 0.25 = 1.25

100 users → 125 → 156 → 195 → 244...
Exponential growth!
```

### Understanding K Values

| K Value | Meaning | Growth Pattern |
|---------|---------|----------------|
| K < 0.5 | Low virality | Slow, decaying |
| K = 0.5-1.0 | Moderate | Helpful boost |
| K = 1.0 | Breakeven | Sustaining |
| K > 1.0 | True viral | Exponential |

**Key Insight:** K > 1 is rare and usually temporary. Most successful companies have K = 0.3-0.7 with strong paid/organic acquisition.

### Viral Cycle Time

K alone doesn't tell the full story. Speed matters.

**Viral Cycle Time (t):** Time from user signup to their invites being accepted.

```
Short cycle (days): Fast compounding
Long cycle (weeks): Slow compounding
```

**Example:**
- Company A: K = 0.8, t = 2 days
- Company B: K = 0.8, t = 14 days

After 30 days from 100 users:
- Company A: ~3,000 users (15 cycles)
- Company B: ~400 users (2 cycles)

**Same K, very different outcomes.**

### Optimizing K Factor

**Increase Invites (i):**
- Prompt at right moment
- Make inviting valuable (collaboration)
- Reduce friction to invite
- Multiple invite mechanisms

**Increase Conversion (c):**
- Compelling invite message
- Personal connection visible
- Incentives for both parties
- Seamless signup flow
- Instant value for invitee

**Dashboard:**


### Virality vs. Other Growth Channels

**Virality Amplifies, Doesn't Replace:**

```
Without virality (K=0):
100 acquired users stay 100

With virality (K=0.5):
100 acquired → 200 total (2x multiplier)

With virality (K=0.8):
100 acquired → 500 total (5x multiplier)

Virality makes every other channel more efficient
```

## Summary: Building Viral Products

### Viral Growth Framework

```
Strong Product Value
        ↓
Natural Sharing Opportunities (usage requires sharing)
        ↓
Low Friction Invites (easy to invite)
        ↓
Great Invited Experience (value for invitee)
        ↓
Invited Users Share
        ↓
[Loop Compounds]
```

### Virality Checklist

**Inherent Virality:**
- [ ] Core usage involves others
- [ ] Product better with more users
- [ ] Sharing is value-add, not spam

**Invite Mechanics:**
- [ ] Easy to invite (multiple methods)
- [ ] Prompted at right moments
- [ ] Compelling default message
- [ ] Incentive for both parties

**Collaboration:**
- [ ] Multi-user features
- [ ] Easy for non-users to view
- [ ] Simple path to account

**Exposure:**
- [ ] Powered-by badges (free tier)
- [ ] Share links include branding
- [ ] Embeds have CTAs
- [ ] Social previews optimized

**Measurement:**
- [ ] K factor calculated
- [ ] Cycle time tracked
- [ ] Invite funnel measured
- [ ] Viral attribution tracked

### Key Metrics

1. **K Factor**: Viral coefficient (target: >0.5)
2. **Cycle Time**: Days for viral cycle (target: <7 days)
3. **Invite Rate**: % users who invite (target: >20%)
4. **Invite Accept**: % invites accepted (target: >25%)
5. **Viral Multiplier**: Total users per acquired user

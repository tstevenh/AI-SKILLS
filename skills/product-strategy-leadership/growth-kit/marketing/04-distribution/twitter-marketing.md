## The Five Viral Formulas

### Formula Selection Guide

| Content Type             | Formula                | Avg. Performance | Trigger Words                          |
| ------------------------ | ---------------------- | ---------------- | -------------------------------------- |
| Disruption, cost savings | R.I.P Formula          | 43K+ likes       | "replaced", "saved $X", "don't need"   |
| Tool comparisons         | I Tested Formula       | 25K+ avg         | "tested", "compared", "vs"             |
| Hidden features          | Sleeping On It Formula | 16K+ avg         | "nobody knows", "hidden", "underrated" |
| Breaking news            | Holy Shit Formula      | 11K+ avg         | "just built", "breakthrough"           |
| Launches, updates        | Alert Formula          | 11K+ avg         | "launching", "just released"           |

### Quick Formula Templates

#### 1. R.I.P Formula (Highest Performer)

**Use When:** Disruption angle, democratization, cost savings

```
R.I.P [Expensive Solution].

You don't need a $[BIG NUMBER] [professional/tool] anymore.

You can now [do X] using [your solution/AI tool].

Here are the exact [#] [prompts/ways] I use to replicate [output] for free:
```

**Why It Works:** Creates shock value, promises specific savings, delivers copy-paste value.

#### 2. I Tested Formula

**Use When:** Comparisons, tool battles, validation

```
I tested [Tool A] and [Tool B] with same critical [test].

The results will blow your mind.

[Tool A]          Vs.          [Tool B]

(Evidence included)
```

**Why It Works:** Leverages curiosity, promises definitive answer, uses visual comparison.

#### 3. Sleeping On It Formula

**Use When:** Hidden features, underutilized capabilities

```
[Tool/Solution] is dangerously good.

But 99% of people are sleeping on what it can actually do.

I've used it to [X], [Y], [Z], and more.

Here are [#] ways to use [tool] that feel like cheating:
```

**Why It Works:** Insider knowledge appeal, exclusivity, promise of unfair advantage.

#### 4. Holy Shit Formula

**Use When:** Breaking news, research announcements, breakthroughs

```
Holy shit. [Organization/Team] just built [revolutionary thing]

It's called [NAME].

Instead of [old method], [NAME] [new approach].

The results?

[Metric 1]
[Metric 2]
[Metric 3]

[Category] that [self-improve] are no longer sci-fi.
```

**Why It Works:** Raw emotion, specific metrics, frames as historic moment.

#### 5. Alert Formula

**Use When:** Announcements, launches, urgent updates

```
[Company/Tool] just did something wild.

[Brief explanation of what happened]

[Why it matters]

Here's what you need to know:
```

**Why It Works:** Urgency, insider knowledge, promise of important information.

## Thread Structure

### Opening Tweet (Hook)

```
[SHOCKING STATEMENT]

[Context that deepens hook]

[Promise of value with number]

[Thread continuation indicator]
```

**Requirements:**

- Under 280 characters (aim for <200)
- Specific number or claim
- Curiosity gap created
- Shocking/controversial element

### Tweet 2 (Setup - Optional)

```
Let me tell you what [traditional approach] actually does:

1. [Step]
2. [Step]
3. [Step]

But guess what?

[New approach] can now do [X%] of that [instantly/for free].

Let me show you how:
```

### Tweets 3-10 (Value Delivery)

```
1/ [Method Name]

[What it does]

Prompt to use:

"[EXACT COPY-PASTE PROMPT]"

[Expected outcome]

---

2/ [Method Name]

[What it does]

Implementation:

"[EXACT STEPS]"

[Real results]
```

**Requirements:**

- Numbered progression
- Copy-paste prompts or steps
- Real metrics where possible
- Line breaks between sections

### Final Tweet (CTA)

```
[Transformation statement]

[Product/Service CTA if applicable]

I hope you've found this helpful.

Follow me @[handle] for more.

Like/Repost if you can
```

## Voice Guidelines

### Technical Credibility + Casual Authority

**DO:**

- Start with shock value ("R.I.P", "Holy shit", "dangerously good")
- Use specific metrics ("$80K saved", "600 invoices", "5-level classification")
- Name real tools and companies when sharing implementations
- Share exact implementations (prompts, architectures, code)
- Balance technical depth with accessibility

**DON'T:**

- Overhype without proof
- Make claims without metrics
- Promote tools without implementation details
- Talk in abstract terms
- Use corporate jargon

### Voice Examples

**Good:**

- "We process 600+ invoices monthly using [tool]. Here's the exact system:"
- "R.I.P $99/month SaaS tools. I replaced 5 subscriptions with one solution."
- "Same architecture. Three industries. Different implementations. Here's what worked everywhere:"

**Bad:**

- "AI is revolutionizing operations" (too vague, no proof)
- "Let me tell you about my journey" (too personal, not tactical)
- "Check out this cool trick" (too casual, no authority)
- "Leveraging synergies through AI transformation" (corporate jargon)

### Forbidden Phrases

- "AI is revolutionizing..."
- "Let me tell you about my journey..."
- "Check out this cool trick..."
- "Leveraging synergies..."
- Any vague claim without specific metric

## Formatting Rules

### Line Breaks (Critical)

```
GOOD:
"Hook line.

Context line.

Promise line."

BAD:
"Hook line. Context line. Promise line."
```

Always put line breaks between distinct thoughts. Create visual breathing room.

### Emoji Usage

```
GOOD:
"Holy shit. [Statement]"
"[Alert statement]"
"[Benefit]"

BAD:
"[Statement]"
```

**Rules:**

- 1-2 strategic emojis per tweet maximum
- Use function emojis (alerts, shock indicators, checkmarks)
- Never decorative spam

### Numbers

**Always specific over ranges:**

- "$80K saved" not "$75K-$85K saved"
- "600+ invoices" not "hundreds of invoices"
- "5-level classification" not "multi-level classification"

### Character Limits

- Tweet maximum: 280 characters
- Aim for hooks under 200 chars (leaves room for engagement)

## Iteration Patterns

When users request changes:

### "Make it punchier"

- Provide 3 alternative hooks using power words
- Suggest starting with R.I.P or Holy Shit formula
- Add specific metrics to opening
- Create stronger curiosity gap

### "More technical depth"

- Add architecture details
- Include tool stack specifics
- Reference technical decisions made
- Add code snippets or implementation steps

### "More accessible"

- Simplify technical jargon
- Add analogies or comparisons
- Focus on outcomes over process
- Use "Here's what this means" explanations

### "Different angle"

- Suggest 2-3 alternative formulas for same content
- Reframe from disruption to capability to comparison
- Offer industry-specific vs. cross-industry framing

## Common Mistakes

| Mistake                         | Fix                                      |
| ------------------------------- | ---------------------------------------- |
| Vague value props ("Some tips") | Specific promise ("Exact 5 prompts")     |
| Burying the lead                | Start with the shocking part             |
| No curiosity gap                | Don't reveal everything in hook          |
| Missing metrics ("Really fast") | Quantify ("10x faster")                  |
| Weak CTAs ("Check this out")    | Strong ("Here's the exact framework")    |
| Too many topics                 | One focused thread beats scattered ideas |
| Abstract without concrete       | Always include real examples/metrics     |

## Golden Rules

1. **Shock value > Politeness** - "R.I.P expensive consultants" not "consultants might not be needed"
2. **Specificity > Vagueness** - "$300k consultant" not "expensive consultant"
3. **Copy-paste value > Explanations** - Exact prompts not "how to write prompts"
4. **Metrics > Claims** - "+40% improvement" not "much better"
5. **Promise then deliver** - Hook promises value, body delivers exactly that

## Engagement Strategy

### Single Tweet Engagement

**High-engagement patterns:**

- Ask questions that invite opinions
- Share hot takes with reasoning
- Post specific results with screenshots
- Quote tweet with added value (not just "This")

### Thread Engagement

**Maximize thread performance:**

- Self-reply to add updates
- Respond to every comment in first hour
- Pin high-value follow-up tweets
- Quote your own thread with new angle

### Reply Strategy

- Reply to relevant threads in your space
- Add genuine value, not self-promotion
- Build relationships with consistent engagement
- 80/20 rule: 80% pure value, 20% soft mentions

## Summary

Twitter marketing with this skill requires:

1. **Formula selection** - Match content type to proven formula

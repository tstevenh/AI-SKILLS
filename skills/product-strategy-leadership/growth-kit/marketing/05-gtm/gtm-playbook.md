# Go-to-Market Playbook

Launch strategy and growth tactics for product releases, addon launches, and major version updates.

---

## GTM Philosophy

**Core Principles:**

- Ship fast, learn fast, iterate fast
- Directory/content builds authority that converts to sales
- Early buyers rewarded with new features (compounding value)
- 70% value content, 30% product content
- Price increases with product expansion (urgency without discounting)

---

## Launch Types

### Type 1: Major Version Launch

**Timeline:** 2-3 weeks preparation
**Channels:** All channels, full push
**Pricing Impact:** May include price increase

### Type 2: New Feature/Addon Launch

**Timeline:** 1 week preparation
**Channels:** Email to existing customers + social + community
**Pricing Impact:** Optional price increase

### Type 3: Minor Update

**Timeline:** Same-day announcement
**Channels:** Email + changelog + social
**Pricing Impact:** None (adds value to existing price)

### Type 4: Content/Directory Expansion

**Timeline:** Ongoing
**Channels:** SEO + social shares of new resources
**Pricing Impact:** None (traffic building)

## Major Version Launch Playbook

### Pre-Launch (2 Weeks Before)

**Week -2:**

- [ ] Finalize feature list and documentation
- [ ] Update all product documentation
- [ ] Prepare comparison content (what's new)
- [ ] Draft announcement email
- [ ] Create social media content calendar
- [ ] Update landing page with new features
- [ ] Record demo/walkthrough if applicable

**Week -1:**

- [ ] Beta test with select existing customers
- [ ] Collect pre-launch testimonials
- [ ] Finalize pricing decision
- [ ] Prepare changelog
- [ ] Schedule all social posts
- [ ] Brief community moderators (if applicable)

### Launch Day

**Morning:**

- [ ] Push product update live
- [ ] Update landing page
- [ ] Publish changelog
- [ ] Send email announcement (existing customers first)

**Afternoon:**

- [ ] Post on Twitter/X (main announcement)
- [ ] Post in community channels
- [ ] Engage with all responses

**Evening:**

- [ ] Monitor for issues
- [ ] Respond to all feedback
- [ ] Thank early upgraders publicly

### Post-Launch (Week 1)

- [ ] Daily: Monitor metrics, respond to feedback
- [ ] Day 2-3: Follow-up email with tips/highlights
- [ ] Day 5-7: Collect and publish new testimonials
- [ ] End of week: Metrics review, iterate messaging

## Feature/Addon Launch Playbook

### Pre-Launch (1 Week Before)

- [ ] Complete feature documentation

### Launch Day

- [ ] Push feature live
- [ ] Update sales page with new pricing (if applicable)
- [ ] Send email: "New [feature type] just dropped - [Name]"
- [ ] Post on social channels
- [ ] Update documentation

### Post-Launch

- [ ] Monitor early usage for issues
- [ ] Collect feedback for iteration
- [ ] Create tutorial content
- [ ] Plan next feature based on learnings

## Email Templates

### New Feature Announcement

**Subject:** New [feature type] just dropped: [Feature Name]

**Body:**

```
Hey [Name],

Quick update: [Feature Name] is now live.

**What it does:**
[2-3 sentence description]

**How to use it:**
[Quick setup instructions]

**[Optional: Pricing update]:**
As promised, the price goes up with each addition. New price is $[X] for new customers. You already have it - no action needed.

Questions? Reply to this email.

[Signature]
```

### Major Version Announcement

**Subject:** [Product] [Version] is here - [Key Feature/Benefit]

**Body:**

```
Hey [Name],

[Version] just shipped. Here's what's new:

**Headlines:**
- [Feature 1]: [Benefit]
- [Feature 2]: [Benefit]
- [Feature 3]: [Benefit]

**Upgrade instructions:**
[How to get the new version]

**Coming next:**
[Tease future development]

As always, you've got lifetime updates - this is included.

[Signature]
```

### Pre-Launch Teaser

**Subject:** Something new coming [Day/Date]

**Body:**

```
Hey [Name],

Been working on something for a while. Shipping it [day/date].

Quick preview:
[One sentence teaser - creates curiosity without full reveal]

Keep an eye on your inbox.

[Signature]
```

## Risk Mitigation

### Launch Risks

| Risk                   | Mitigation                                    |
| ---------------------- | --------------------------------------------- |
| Product bugs at launch | Beta test, have rollback ready                |
| Low launch engagement  | Pre-build email list, coordinate community    |
| Negative feedback      | Respond quickly, fix fast, be transparent     |
| Competitor timing      | Focus on own roadmap, don't react             |
| Technical issues       | Test checkout, delivery, access before launch |

### Communication Risks

| Risk                     | Mitigation                              |
| ------------------------ | --------------------------------------- |
| Message doesn't resonate | A/B test subject lines, iterate quickly |
| Wrong audience reached   | Segment email list, target content      |
| Over-promising           | Under-promise, over-deliver on features |

## Pricing Strategy Framework

### Price Increase Triggers

- New major feature/addon added
- Significant value improvement
- Market positioning shift
- Anniversary/milestone

### Price Increase Communication

**To existing customers:**

> "Good news: [New feature] is now included in your purchase at no extra cost. For new customers, the price is now $[X] to reflect the additional value."

**To prospects:**

> "Price increases [date] when [reason]. Lock in current pricing before then."

### Urgency Without Discounting

- Price increases with each addon (natural urgency)
- Limited-time bonuses vs. discounts
- Early adopter perks (first to test new features)
- Grandfathered pricing for early customers

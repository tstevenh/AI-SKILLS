# Content Creation SOP

## Purpose

This SOP defines the standard process for creating content for guest posts and other link building purposes. Following this process ensures consistent quality and efficient production.

## Scope

Applies to: Content writers, editors, and link builders creating content for link building campaigns.

## Process Overview

```
1. Receive Brief → 2. Research → 3. Outline → 4. Write → 5. Self-Edit → 6. Review → 7. Submit
```

## Step 1: Receive and Review Brief

### Required Brief Information

Before starting any content:

**Campaign details:**
- Client/campaign name
- Target site URL
- Submission guidelines URL
- Deadline

**Content requirements:**
- Topic/title (approved)
- Word count
- Link specifications (URL, anchor text)
- Specific requirements from site

**Reference materials:**
- Site voice/style examples
- Brand guidelines
- Previous successful articles
- Keywords (if applicable)

### Brief Validation

Verify before proceeding:
- [ ] All required information present
- [ ] Deadline is achievable
- [ ] Topic is clear and approved
- [ ] Link specifications are clear
- [ ] No conflicting requirements

**If anything is unclear, clarify before proceeding.**

## Step 2: Research

### Site Research

**Understand the target site:**
1. Read site's "About" page
2. Review 3-5 recent articles
3. Note voice and style
4. Check formatting conventions
5. Review their guidelines again

**Document observations:**
- Formal vs. casual tone
- First person vs. third person
- Paragraph length preference
- Use of headers
- Image requirements

### Topic Research

**Gather information:**
1. Search for existing coverage of topic
2. Find supporting data and statistics
3. Identify expert quotes (if applicable)
4. Note examples and case studies
5. Understand the target audience

**Research checklist:**
- [ ] Understand topic thoroughly
- [ ] Have supporting data/sources
- [ ] Know what makes this unique
- [ ] Understand reader's perspective
- [ ] Know the key takeaways

## Step 3: Create Outline

### Outline Template

```
# [TITLE]

## Introduction (150-200 words)
- Hook: [Attention-grabbing opening]
- Context: [Why this matters]
- Preview: [What reader will learn]

## [Section 1 Title] (300-400 words)
- Key point: [Main idea]
- Supporting evidence: [Data/example]
- [LINK OPPORTUNITY if appropriate]

## [Section 2 Title] (300-400 words)
- Key point: [Main idea]
- Supporting evidence: [Data/example]

## [Section 3 Title] (300-400 words)
- Key point: [Main idea]
- Supporting evidence: [Data/example]
- [LINK OPPORTUNITY if appropriate]

## [Section 4 Title] (300-400 words)
- Key point: [Main idea]
- Supporting evidence: [Data/example]

## Conclusion (150-200 words)
- Summary: [Key points]
- Takeaway: [Main message]
- Call to action: [What reader should do]

## Author Bio
[Bio as specified]
```

### Outline Quality Check

Before writing:
- [ ] Logical flow of ideas
- [ ] All key points covered
- [ ] Link placement(s) identified
- [ ] Meets word count estimate
- [ ] Aligns with site's content

## Step 4: Write Content

### Writing Standards

**Introduction:**
- Hook in first sentence
- Establish relevance quickly
- Tell reader what they'll learn
- No longer than 200 words

**Body sections:**
- One main idea per section
- Clear topic sentences
- Supporting evidence for claims
- Practical, actionable content
- Smooth transitions

**Conclusion:**
- Summarize key points
- Clear takeaway message
- Call to action if appropriate
- No new information

### Writing Quality Guidelines

**Clarity:**
- Short sentences (average 15-20 words)
- Short paragraphs (2-4 sentences)
- Active voice
- No jargon without explanation

**Value:**
- Every section adds value
- Specific, not generic
- Actionable insights
- Reader-focused

**Originality:**
- Fresh perspective
- Unique insights
- Not rehashed content
- Your own words

### Link Integration

**Natural placement:**
- Link flows naturally in sentence
- Surrounding content is relevant
- Not in first paragraph
- Not forced or awkward

**Anchor text:**
- Uses approved anchor
- Reads naturally
- Not over-optimized
- Fits sentence structure

**Example good integration:**
"When planning your content calendar, using a project management tool like [anchor text linking to target] can help keep your team organized and deadlines on track."

**Example poor integration:**
"Content planning is important. [Best project management software] helps with this."

### Internal Links

**Include 2-3 internal links to their content:**
- Find relevant articles on their site
- Link naturally within your content
- Use descriptive anchor text
- Space throughout article

## Step 5: Self-Edit

### First Pass: Content Review

**Read through for:**
- [ ] Logical flow and structure
- [ ] Completeness of coverage
- [ ] Value and insights
- [ ] Accuracy of information
- [ ] Link integration quality

### Second Pass: Line Edit

**Check for:**
- [ ] Grammar and spelling
- [ ] Sentence clarity
- [ ] Word choice
- [ ] Redundancy
- [ ] Transitions

### Third Pass: Format Review

**Verify:**
- [ ] Headers are clear and descriptive
- [ ] Paragraphs are appropriate length
- [ ] Lists are used where helpful
- [ ] Formatting is consistent
- [ ] Word count meets requirement

### Final Check

**Before submitting for review:**
- [ ] All facts verified with sources
- [ ] All links working
- [ ] Images included (if required)
- [ ] Author bio attached
- [ ] Meets all guidelines

## Step 6: Peer/Editor Review

### Review Process

**Reviewer checks:**
1. Content quality and value
2. Alignment with guidelines
3. Link integration
4. Writing quality
5. Formatting

**Review outcomes:**
- Approved: Ready for submission
- Minor revisions: Small changes needed
- Major revisions: Significant rewrite needed
- Reject: Start over (rare)

### Handling Feedback

**When receiving feedback:**
- Review all comments
- Ask clarifying questions if needed
- Make requested changes
- Re-submit for review if major changes

**Revision timeline:**
- Minor revisions: Same day
- Major revisions: 1-2 days

## Step 7: Submit

### Submission Preparation

**Final package includes:**
- Content document (clean, formatted)
- Featured image (if required)
- Author headshot (if required)
- Author bio (if separate)
- Any other required materials

### Submission Method

**Follow site's specific process:**
- Email to specified contact
- Submission form
- Google Doc shared
- CMS direct entry

### Post-Submission

**After submitting:**
- [ ] Confirm submission receipt
- [ ] Note any revisions requested
- [ ] Complete revisions promptly
- [ ] Confirm publication date
- [ ] Verify live once published

## Quality Standards

### Content Quality Requirements

**Minimum standards:**
- 100% original content
- No plagiarism
- No AI-generated spam
- Grammatically correct
- Provides genuine value

**Quality targets:**
- Quality score: 4.0+ (on 5-point scale)
- Revision rate: <30%
- First-time acceptance: 70%+
- Publisher satisfaction: High

### Quality Scoring Guide

| Score | Description |
|-------|-------------|
| 5.0 | Exceptional - Publisher excited to feature |
| 4.5 | Excellent - Above expectations |
| 4.0 | Good - Meets quality standards |
| 3.5 | Acceptable - Minor improvements possible |
| 3.0 | Marginal - Needs work |
| <3.0 | Unacceptable - Reject/rewrite |

## Common Mistakes to Avoid

### Content Mistakes

- [ ] NOT thin content (<1000 meaningful words)
- [ ] NOT rehashed generic advice
- [ ] NOT keyword-stuffed
- [ ] NOT promotional/salesy
- [ ] NOT off-topic tangents

### Writing Mistakes

- [ ] NOT passive voice throughout
- [ ] NOT walls of text
- [ ] NOT confusing structure
- [ ] NOT inconsistent voice
- [ ] NOT grammar/spelling errors

### Link Mistakes

- [ ] NOT forced anchor text
- [ ] NOT link in first paragraph
- [ ] NOT over-linking (1-2 max)
- [ ] NOT irrelevant link context
- [ ] NOT broken links

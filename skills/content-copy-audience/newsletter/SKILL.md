# Newsletter

This skill covers two jobs:

1. write a newsletter worth reading
2. run a newsletter worth scaling

## Start with the right mode

Choose one primary mode first.

### Editorial mode

Use for:

- writing a newsletter issue
- choosing a format
- improving voice and structure
- creating recurring sections

### Operator mode

Use for:

- growth planning
- monetization
- sponsor operations
- platform selection
- analytics and optimization

## Editorial mode

Choose the core format before drafting.

### Common newsletter archetypes

- Deep-dive / framework
- News briefing
- Curated links + commentary
- Personal essay / reflection
- Builder or startup update
- Irreverent business/news format

For each issue, define:

1. the one idea or theme
2. the audience segment
3. the reason this matters now
4. the action or feeling you want after reading

## Writing workflow

Use this default sequence:

`ANGLE -> SUBJECT -> HOOK -> STRUCTURE -> VALUE -> CTA -> EDIT`

### Subject line

Subject lines should do one of three things:

- promise value
- create curiosity
- signal a useful update

### Hook

Open fast. A hook should create immediate relevance:

- a sharp observation
- a timely event
- a contrarian claim
- a personal moment with a clear lesson

### Structure

Strong newsletters are scannable:

- short paragraphs
- clear section headers
- bolded takeaways where useful
- one dominant arc, not five unrelated ideas

### CTA

Pick one primary CTA:

- reply
- share
- click
- buy
- book
- join

## Operator mode

### Growth levers

Default priority:

1. referral loops
2. cross-promotions
3. welcome-sequence optimization
4. social-to-email conversion
5. SEO or content flywheels
6. paid acquisition, only after baseline funnel health is proven

### Monetization stack

Common models:

- sponsorships
- paid subscriptions
- affiliate revenue
- products or courses
- community memberships
- services or advisory

Pick the model that matches the newsletter type. Do not force sponsorships onto a tiny list with weak engagement.

### Metrics that matter

Track at minimum:

- open rate
- click rate
- monthly list growth
- unsubscribe rate
- revenue per subscriber
- paid churn if subscriptions exist

Use these metrics to diagnose:

- low opens -> subject line, deliverability, sender reputation
- low clicks -> weak angle, poor structure, bad CTA
- low growth -> acquisition problem
- high churn -> audience mismatch or monetization mismatch

### Weekly operator cadence

Default weekly loop:

1. review metrics
2. plan the next issue
3. draft and edit
4. schedule and send
5. monitor performance
6. log what to keep, cut, or test next week

## Platform selection

Use rough-fit selection:

- Beehiiv for growth-led free newsletters
- Kit / ConvertKit for creator monetization and automations
- Substack for writer-first paid subscription models
- Ghost for teams that want more ownership and customization

## Deliverable format

Return one of these, depending on the request:

### For writing requests

1. format choice
2. subject line options
3. full issue draft
4. CTA options
5. optional send-day or testing note

### For strategy requests

1. newsletter type and business model fit
2. growth diagnosis
3. monetization recommendations
4. platform recommendation
5. 30-day improvement plan

## Rule of thumb

A good newsletter is not "content sent by email."

It is a repeatable product with:

- a recognizable point of view
- a dependable format
- a measurable business loop behind it

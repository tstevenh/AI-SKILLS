# Adversarial QA

Comprehensive testing and breaking strategy for web applications, APIs, and digital products.

## What It Does

This skill enables systematic adversarial quality assurance testing—actively attempting to break systems, discover edge cases, exploit vulnerabilities, and identify failure modes that escape conventional testing.

## Features

- Red-teaming strategies and chaos engineering
- Fault injection patterns and edge case discovery
- Web application security testing checklists
- Funnel and conversion flow breaking techniques
- API adversarial testing methods
- Input validation attack patterns
- WordPress-specific attack vectors
- OWASP Top 10 testing simplified

## Installation

1. Copy the `adversarial-qa/` folder into your OpenClaw skills directory
2. Restart OpenClaw

## Usage

Use this skill when:
- Testing web applications for vulnerabilities
- Performing security audits
- Conducting chaos engineering experiments
- Testing form validation and error handling
- Checking rate limiting and throttling
- Validating input sanitization

## Requirements

- **WARNING**: Only use on systems you have explicit written authorization to test
- Testing tools (Burp Suite, OWASP ZAP, or similar)
- Command line tools (curl, wget, nmap)
- Python/Node.js for automation scripts
- Staging environment for safe testing

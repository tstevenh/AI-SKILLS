# Functionality QA

Complete testing methodologies for ensuring software quality and reliability.

## What It Does

This skill covers functional testing approaches—from test case design and E2E testing to API validation and WordPress-specific testing. Ensure your applications work correctly under all conditions.

## Features

- Test case design methodologies (equivalence partitioning, BVA)
- End-to-end testing workflows
- API testing and validation
- Database integrity testing
- Form validation testing
- Session and cookie management testing
- OAuth and SSO flow testing
- Payment gateway testing
- WordPress-specific testing patterns

## Installation

1. Copy the `functionality-qa/` folder into your OpenClaw skills directory
2. Restart OpenClaw

## Usage

Use this skill when:
- Writing test cases for new features
- Performing E2E testing
- Testing API endpoints
- Validating form submissions
- Testing authentication flows
- Checking payment processing
- Running regression test suites

## Requirements

- Testing framework (Selenium, Cypress, Playwright, or Puppeteer)
- API testing tool (Postman, Insomnia, or curl)
- Access to test environments
- Test data and fixtures
- Bug tracking system access

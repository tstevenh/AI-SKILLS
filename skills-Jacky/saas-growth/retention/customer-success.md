# Customer Success

## Introduction

Customer Success (CS) is the proactive effort to ensure customers achieve their desired outcomes using your product. Unlike reactive support, CS anticipates needs, guides customers toward value, and intervenes before problems become cancellations. For SaaS companies, CS is often the difference between good and great retention.

---

## 1. When to Add Customer Success

### Signals You Need CS

**Revenue Signals:**
- Churn exceeding benchmarks
- Expansion revenue flat
- Renewals requiring effort

**Customer Signals:**
- Customers not achieving value
- Low product adoption
- Feedback of being "lost"

**Business Model Signals:**
- ACV > $5K (justifies touch)
- Complex implementation
- Multi-stakeholder customers

### CS Before You're Ready

**Don't hire CS too early:**
- Before product-market fit
- When product is the problem
- Before you understand churn reasons

**Instead:**
- Founders do CS initially
- Learn what customers need
- Build CS motions before hiring

## 2. CS Team Structure

### CS Roles

**Customer Success Manager (CSM):**
- Owns customer relationship
- Drives adoption and outcomes
- Manages renewals
- Identifies expansion

**Implementation/Onboarding:**
- New customer setup
- Data migration
- Training
- Time to value

**Technical Support:**
- Break/fix issues
- Technical questions
- Documentation
- Ticket resolution

**CS Operations:**
- Health scores
- Reporting
- Process design
- Tool management

### Team Sizing

**CSM Ratios:**

| Model | Customers per CSM |
|-------|-------------------|
| High-touch Enterprise | 10-20 |
| Mid-market | 30-75 |
| SMB Touch | 100-300 |
| Tech-touch | 500-2000+ |

**Calculating CSM Headcount:**
```
CSMs Needed = Total Customers / Customers per CSM

Example:
- 500 mid-market customers
- 50 customers per CSM
- Need: 10 CSMs
```

## 3. CS Motions

### High-Touch CS

**For:** Enterprise, high ACV

**Activities:**
- Dedicated CSM
- Regular check-ins (weekly/biweekly)
- QBRs (Quarterly Business Reviews)
- Executive sponsors
- Custom success plans

**CSM Activities:**
| Activity | Frequency |
|----------|-----------|
| Check-in call | Weekly/Biweekly |
| QBR | Quarterly |
| Health review | Monthly |
| Renewal planning | 90 days out |

### Low-Touch CS

**For:** SMB, lower ACV

**Activities:**
- Pooled CSM coverage
- Reactive engagement
- Automated outreach
- Community support
- Self-serve resources

### Tech-Touch CS

**For:** Self-serve, volume

**Activities:**
- Automated emails
- In-app guidance
- Self-serve resources
- Community
- Triggered outreach

**Tech-Touch Triggers:**
- No login in X days
- Feature not used
- Usage declining
- Support ticket
- Renewal approaching

## 4. Success Playbooks

### Onboarding Playbook

**Goal:** Get customer to first value ASAP

**Timeline Example (30-day):**

| Day | Activity | Owner |
|-----|----------|-------|
| 0 | Welcome call | CSM |
| 1-3 | Account setup | Customer + CSM |
| 7 | Check-in call | CSM |
| 14 | Adoption review | CSM |
| 21 | Training session | CSM |
| 30 | Success review | CSM |

### Adoption Playbook

**Goal:** Drive feature adoption

**Activities:**
- Feature education emails
- Adoption scorecards
- Training webinars
- Use case guidance

### Renewal Playbook

**Goal:** Secure renewal

**Timeline (90 days out):**

| Days Out | Activity |
|----------|----------|
| 90 | Health assessment |
| 60 | Renewal conversation |
| 45 | Proposal/quote |
| 30 | Negotiation |
| 14 | Contract finalization |
| 0 | Renewal complete |

### Expansion Playbook

**Goal:** Grow account value

**Triggers:**
- Usage approaching limits
- New use cases emerging
- Team growth
- Positive NPS

**Activities:**
- Expansion conversation
- ROI review
- Proposal
- Close

## 5. QBRs and Check-ins

### Quarterly Business Review (QBR)

**Purpose:**
- Review value delivered
- Align on goals
- Identify expansion
- Strengthen relationship

**QBR Agenda:**
```
1. Wins & Value Delivered (10 min)
   - Metrics achieved
   - Success stories
   
2. Adoption & Usage Review (10 min)
   - Usage data
   - Feature adoption
   
3. Challenges & Feedback (10 min)
   - Pain points
   - Product feedback
   
4. Goals for Next Quarter (15 min)
   - Customer goals
   - How we can help
   
5. Product Roadmap (10 min)
   - Upcoming features
   - Alignment with needs
   
6. Next Steps (5 min)
   - Action items
   - Follow-up timeline
```

### Effective Check-ins

**Good Check-in:**
- Prepared with data
- Agenda shared beforehand
- Value-focused
- Action items documented

**Bad Check-in:**
- "Just checking in"
- No agenda
- Wasting customer time
- No follow-through

## Summary: CS Framework

### Building CS Function

**Phase 1: Founder-led**
- Founders own relationships
- Learn what customers need
- Document patterns

**Phase 2: First hire**
- Hire CSM
- Define processes
- Build playbooks

**Phase 3: Scale**
- Add team members
- Specialize roles
- Segment approach

### CS Checklist

- [ ] CS coverage model defined
- [ ] Health scoring implemented
- [ ] Onboarding playbook created
- [ ] Renewal process documented
- [ ] QBR template ready
- [ ] Expansion triggers identified
- [ ] CS metrics tracked

### CS Metrics

| Metric | Target |
|--------|--------|
| Gross Retention | >90% |
| Net Retention | >100% |
| NPS | >40 |
| Time to Value | <30 days |
| Renewal Rate | >85% |
| Expansion Rate | >20% |

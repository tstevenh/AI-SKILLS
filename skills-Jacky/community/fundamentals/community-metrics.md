# Community Metrics: The Complete Measurement Guide

## Overview

What gets measured gets managed. This guide covers every metric community builders need to track, from acquisition through retention and monetization, with benchmarks and actionable insights.

---

## Part 1: Metrics Framework

### The Metric Hierarchy

**Level 1: North Star Metric**
```
One metric that matters most to success
Examples:
- Monthly Active Engaged Members
- Net Revenue Retention
- Member Transformation Rate
```

**Level 2: Core Metrics (3-5)**
```
Metrics that directly drive North Star
Examples:
- Acquisition rate
- Engagement rate
- Retention rate
- Revenue per member
```

**Level 3: Supporting Metrics (5-10)**
```
Metrics that support core metrics
Examples:
- Trial conversion rate
- Event attendance
- Content engagement
- NPS score
```

**Level 4: Diagnostic Metrics (Many)**
```
Detailed metrics for troubleshooting
Examples:
- Post engagement by type
- Churn by cohort
- Login frequency
- Feature usage
```

### Metrics Categories

**Growth Metrics**: How fast are we growing?
**Engagement Metrics**: How active are members?
**Retention Metrics**: How well do we keep members?
**Revenue Metrics**: How much are we making?
**Satisfaction Metrics**: How happy are members?

## Part 2: Growth Metrics

### Member Growth Metrics

**Total Members**
```
Definition: Total number of members at any point
Formula: Previous total + new - churned
Track: Monthly, quarterly
```

**New Members**
```
Definition: Members who joined in period
Formula: Count of new signups
Track: Weekly, monthly
```

**Growth Rate**
```
Definition: Rate of membership growth
Formula: (New members - Churned) / Previous total × 100
Benchmark: 5-10% monthly is healthy
Track: Monthly
```

**Net Member Change**
```
Definition: Net change in membership
Formula: New members - Churned members
Track: Monthly
Ideal: Positive and growing
```

### Acquisition Metrics

**Traffic to Signup Rate**
```
Definition: Visitors who become members
Formula: Signups / Unique visitors × 100
Benchmark: 2-5% for free, 0.5-2% for paid
```

**Trial to Paid Conversion**
```
Definition: Trial members who convert to paid
Formula: Paid conversions / Trial starts × 100
Benchmark: 30-50% is excellent
```

**Source Attribution**
```
Track members by source:
- Organic search
- Social media
- Email
- Referrals
- Paid ads
- Other
```

**Cost Per Acquisition (CPA)**
```
Definition: Cost to acquire one member
Formula: Total acquisition spend / New members
Benchmark: Should be < 1/4 of LTV
```

### Referral Metrics

**Referral Rate**
```
Definition: Members who refer others
Formula: Referring members / Total members × 100
Benchmark: 10-20% is excellent
```

**Viral Coefficient**
```
Definition: Members created per existing member
Formula: (Referrals × Conversion rate) / Total members
Benchmark: >1 means viral growth
```

**Referral Revenue**
```
Definition: Revenue from referrals
Formula: Revenue from referred members
Track: % of total revenue from referrals
```

## Part 3: Engagement Metrics

### Activity Metrics

**Daily Active Users (DAU)**
```
Definition: Unique members active in a day
Formula: Count of unique active members daily
Benchmark: Varies by community type
```

**Weekly Active Users (WAU)**
```
Definition: Unique members active in a week
Formula: Count of unique active members in 7 days
Benchmark: 30-50% of total members
```

**Monthly Active Users (MAU)**
```
Definition: Unique members active in a month
Formula: Count of unique active members in 30 days
Benchmark: 50-70% of total members
```

**Stickiness Ratio**
```
Definition: How often active users return
Formula: DAU / MAU × 100
Benchmark: >20% is good, >40% is excellent
```

**Sessions per Member**
```
Definition: Average visits per member
Formula: Total sessions / Active members
Track: Weekly average
```

### Content Engagement

**Posts per Member**
```
Definition: Content creation rate
Formula: Total posts / Active members
Benchmark: 0.5-2 posts/member/month
```

**Comments per Post**
```
Definition: Discussion depth
Formula: Total comments / Total posts
Benchmark: 5-15 comments/post is healthy
```

**Engagement Rate**
```
Definition: Interactions per content
Formula: (Likes + Comments + Shares) / Post × 100
Benchmark: 5-15% is good
```

**Content Consumption**
```
Track:
- Course completion rates
- Resource downloads
- Video watch time
- Article reads
```

### Event Engagement

**Event Attendance Rate**
```
Definition: Registered attendees who show
Formula: Attendees / Registrations × 100
Benchmark: 40-60% for virtual
```

**Event Participation Rate**
```
Definition: Attendees who participate
Formula: Participants (chat, Q&A) / Attendees × 100
Benchmark: 30-50% participation
```

**Event NPS**
```
Definition: Satisfaction with events
Formula: (Promoters - Detractors) / Total × 100
Benchmark: >30 is good, >50 is excellent
```

### Engagement Segmentation

**Engagement Tiers**
```
HIGHLY ENGAGED (Super users):
- Top 5-10% of activity
- Multiple posts/week
- High event attendance

ACTIVELY ENGAGED:
- Regular participation
- Weekly activity
- Occasional posts

PASSIVELY ENGAGED:
- Consumes content
- Rarely posts
- Occasional login

DORMANT:
- No recent activity
- Still subscribed
- At-risk

CHURNED:
- Cancelled or left
```

## Part 4: Retention Metrics

### Churn Metrics

**Monthly Churn Rate**
```
Definition: Members lost in a month
Formula: Churned members / Start of month members × 100
Benchmark: <5% monthly is good
```

**Annual Churn Rate**
```
Definition: Members lost in a year
Formula: 1 - (1 - Monthly churn)^12
Example: 5% monthly = 46% annual
```

**Voluntary vs Involuntary Churn**
```
Voluntary: Member chooses to cancel
Involuntary: Payment failure, etc.
Track: Separately to address differently
```

### Retention Metrics

**Member Retention Rate**
```
Definition: Members who stay
Formula: 100% - Churn rate
Or: Ending members / Starting members × 100
```

**Cohort Retention**
```
Track retention by signup cohort:
- Month 1 retention: X%
- Month 3 retention: X%
- Month 6 retention: X%
- Month 12 retention: X%
```

**Member Lifetime**
```
Definition: Average time member stays
Formula: 1 / Monthly churn rate
Example: 5% churn = 20 months average
```

### Renewal Metrics

**Renewal Rate**
```
Definition: Members who renew at term end
Formula: Renewed / Up for renewal × 100
Benchmark: >70% for annual
```

**Save Rate**
```
Definition: At-risk members retained
Formula: Saved members / Cancellation attempts × 100
Track: Effectiveness of retention efforts
```

## Part 5: Revenue Metrics

### Core Revenue Metrics

**Monthly Recurring Revenue (MRR)**
```
Definition: Predictable monthly revenue
Formula: Paying members × Average price
Track: Monthly, trend over time
```

**Annual Recurring Revenue (ARR)**
```
Definition: Annualized recurring revenue
Formula: MRR × 12
Track: Quarterly, annual
```

**Average Revenue Per User (ARPU)**
```
Definition: Revenue per member
Formula: Total revenue / Total members
Track: Monthly, by segment
```

### Growth Revenue Metrics

**New MRR**
```
Definition: Revenue from new members
Formula: Sum of new member payments
Track: Monthly
```

**Expansion MRR**
```
Definition: Revenue from upgrades
Formula: Sum of upgrade payments
Track: Monthly
```

**Contraction MRR**
```
Definition: Revenue lost from downgrades
Formula: Sum of downgrade reductions
Track: Monthly
```

**Churned MRR**
```
Definition: Revenue lost from cancellations
Formula: Sum of cancelled member payments
Track: Monthly
```

**Net MRR Growth**
```
Definition: Overall MRR change
Formula: New + Expansion - Contraction - Churned
Track: Monthly, should be positive
```

### Profitability Metrics

**Customer Lifetime Value (LTV)**
```
Definition: Total revenue from a member
Formula: ARPU × Member lifetime
Or: ARPU / Monthly churn rate
```

**LTV:CAC Ratio**
```
Definition: Value vs acquisition cost
Formula: LTV / Customer Acquisition Cost
Benchmark: >3:1 is healthy
```

**Gross Margin**
```
Definition: Revenue minus direct costs
Formula: (Revenue - Direct costs) / Revenue × 100
Benchmark: >70% for communities
```

### Revenue Retention Metrics

**Gross Revenue Retention (GRR)**
```
Definition: Revenue retained excluding expansion
Formula: (Start MRR - Contraction - Churn) / Start MRR × 100
Benchmark: >85% is good
```

**Net Revenue Retention (NRR)**
```
Definition: Revenue retained including expansion
Formula: (Start MRR + Expansion - Contraction - Churn) / Start MRR × 100
Benchmark: >100% means growth from existing
```

## Part 6: Satisfaction Metrics

### Survey Metrics

**Net Promoter Score (NPS)**
```
Definition: Likelihood to recommend
Question: "How likely are you to recommend?"
Scale: 0-10
Formula: % Promoters (9-10) - % Detractors (0-6)
Benchmark: >30 good, >50 excellent
```

**Customer Satisfaction (CSAT)**
```
Definition: Satisfaction with experience
Question: "How satisfied are you?"
Scale: 1-5
Formula: % selecting 4-5
Benchmark: >80% is good
```

**Customer Effort Score (CES)**
```
Definition: Ease of getting value
Question: "How easy was it to...?"
Scale: 1-7
Formula: % selecting 5-7
Benchmark: >70% is good
```

### Community Health Indicators

**Community Sentiment**
```
Track:
- Positive vs negative posts
- Complaint frequency
- Praise frequency
- Support ticket volume
```

**Member Feedback**
```
Track:
- Feature requests
- Testimonials given
- Reviews written
- Referrals made
```

### Transformation Metrics

**Outcome Achievement**
```
Definition: Members achieving stated goals
Formula: Members achieving / Total members × 100
Track: By goal type, cohort
```

**Progress Metrics**
```
Track progress toward outcomes:
- Milestones completed
- Skills developed
- Goals achieved
- Transformations realized
```

## Part 7: Benchmarks by Community Type

### B2B Professional Community

**Target Benchmarks**:
```
GROWTH:
- Monthly growth rate: 3-8%
- Trial conversion: 30-50%
- Referral rate: 15-25%

ENGAGEMENT:
- WAU/MAU: 40-60%
- Posts per member: 1-3/month
- Event attendance: 50-70%

RETENTION:
- Monthly churn: 3-5%
- NRR: 100-110%
- Annual renewal: 70-85%

REVENUE:
- ARPU: $50-300/month
- LTV: $1,000-5,000
- LTV:CAC: 4:1+
```

### B2C Hobby/Interest Community

**Target Benchmarks**:
```
GROWTH:
- Monthly growth rate: 5-15%
- Conversion rate: 2-5%
- Referral rate: 20-35%

ENGAGEMENT:
- WAU/MAU: 30-50%
- Posts per member: 2-5/month
- Event attendance: 30-50%

RETENTION:
- Monthly churn: 5-8%
- NRR: 90-105%
- Annual renewal: 60-75%

REVENUE:
- ARPU: $20-100/month
- LTV: $200-1,000
- LTV:CAC: 3:1+
```

### Course + Community

**Target Benchmarks**:
```
GROWTH:
- Course completion: 30-60%
- Community adoption: 50-70%
- Referral rate: 10-20%

ENGAGEMENT:
- Weekly active: 40-60%
- Support requests: <5/student
- Event attendance: 50-70%

RETENTION:
- Course churn: <10%
- Community continuation: 30-50%
- Upsell conversion: 10-20%

REVENUE:
- ARPU: Course price + community
- LTV: 1.5-2x course price
- LTV:CAC: 3:1+
```

## Part 9: Metrics Best Practices

### Measurement Principles

**1. Track Trends, Not Snapshots**
```
Single data points mislead
Look at direction over time
Compare to previous periods
```

**2. Segment Your Data**
```
Break down by:
- Cohort (signup date)
- Plan type
- Engagement level
- Source
- Geography
```

**3. Focus on Actionable Metrics**
```
Good metric: Can you change it?
Vanity metric: Makes you feel good but doesn't drive action
```

**4. Set Realistic Benchmarks**
```
Industry benchmarks are starting points
Your historical data matters more
Improve incrementally
```

**5. Review Regularly**
```
Daily: Quick activity check
Weekly: Core metrics review
Monthly: Full analysis
Quarterly: Strategic assessment
```

### Common Mistakes

**Mistake 1: Tracking Too Many Metrics**
```
Problem: Analysis paralysis
Solution: Focus on 5-10 core metrics
```

**Mistake 2: Ignoring Cohort Analysis**
```
Problem: Missing trends in aggregates
Solution: Always look at cohort data
```

**Mistake 3: Vanity Metrics Focus**
```
Problem: Total members up, revenue flat
Solution: Focus on engaged, paying members
```

**Mistake 4: No Baseline**
```
Problem: Can't measure improvement
Solution: Establish baseline before changes
```

**Mistake 5: Reactive Not Proactive**
```
Problem: Only look when problems arise
Solution: Regular scheduled reviews
```

## Summary

### Essential Metrics Checklist

```
MUST TRACK (Core):
□ MRR / Revenue
□ Churn rate
□ Monthly Active Users
□ New member acquisition
□ Engagement rate

SHOULD TRACK (Important):
□ NPS
□ Cohort retention
□ Referral rate
□ LTV
□ Event attendance

NICE TO TRACK (Helpful):
□ Content engagement by type
□ Feature usage
□ Support ticket volume
□ Sentiment analysis
□ Detailed cohort analysis
```

### Metrics Success Formula

```
Right Metrics + Consistent Tracking + Regular Review + Action on Insights = Community Growth
```

### The One Metric Rule

```
If you could only track ONE metric, make it:

ENGAGED MONTHLY ACTIVE MEMBERS

This captures:
- Acquisition (need members to be active)
- Engagement (must be engaged to count)
- Retention (must still be there)

When this grows, everything else follows.
```

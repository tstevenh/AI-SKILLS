# Community Launch Checklist

## Overview

This comprehensive checklist ensures nothing falls through the cracks during your community launch. Use it as a tracking document from pre-launch through post-launch stabilization.

---

## Phase 1: Concept Validation (8-6 Weeks Before)

### Market Validation
- [ ] Defined target audience clearly
- [ ] Identified core problem you're solving
- [ ] Researched competitor communities
- [ ] Surveyed potential members (minimum 20 responses)
- [ ] Conducted 5-10 1:1 validation calls
- [ ] Confirmed willingness to pay at target price

### Community Concept
- [ ] Named the community
- [ ] Written community mission statement
- [ ] Defined transformation/outcome for members
- [ ] Outlined 3-5 core value propositions
- [ ] Created initial positioning statement

### Business Model
- [ ] Determined pricing model (monthly/annual/tiered)
- [ ] Set founding member pricing
- [ ] Calculated target member count for sustainability
- [ ] Projected Year 1 revenue
- [ ] Identified upsell opportunities

---

## Phase 2: Platform Setup (6-4 Weeks Before)

### Platform Selection
- [ ] Evaluated platform options (Skool, Circle, Discord, etc.)
- [ ] Selected platform based on needs
- [ ] Created account and chosen plan
- [ ] Connected payment processing (Stripe)

### Basic Configuration
- [ ] Set community name
- [ ] Uploaded logo
- [ ] Set brand colors
- [ ] Added cover image
- [ ] Written community description
- [ ] Configured custom URL/domain (if applicable)

### Structure Setup
- [ ] Created categories/spaces/channels
- [ ] Set permissions for each area
- [ ] Configured member roles
- [ ] Set up welcome flow
- [ ] Tested navigation on mobile

### Payments
- [ ] Connected Stripe/payment processor
- [ ] Created pricing plans
- [ ] Set up founding member pricing
- [ ] Tested complete purchase flow
- [ ] Configured receipts/invoices

---

- [ ] Templates/tools uploaded

### Launch Content
- [ ] Waitlist page created

## Phase 5: Founding Member Recruitment (2-1 Weeks Before)

### Founding Member Outreach

- [ ] Personal outreach messages sent

- [ ] Issues fixed

## Emergency Protocols

### If Payment Processing Fails
1. Communicate to affected members immediately
2. Offer alternative payment method
3. Document affected transactions
4. Process manually if needed
5. Fix issue urgently

### If Platform Goes Down
1. Check platform status
2. Communicate via email/social
3. Extend deadlines if needed
4. Have backup plan ready

### If Low Sign-ups
1. Don't panic
2. Extend launch window
3. Increase email frequency
4. Personal outreach to warm leads
5. Review messaging

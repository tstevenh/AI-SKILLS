# The Complete Community Launch Playbook

## Overview

This playbook covers everything you need to execute a successful community launch—from the final preparations to the critical first week and beyond.

---

## Part 1: Launch Models

### Open/Close (Enrollment Windows)

**How It Works**:
- Community opens for limited time
- Closes to new members
- Creates urgency and scarcity
- Repeat 2-4 times per year

**Best For**:
- Course-based communities
- High-ticket memberships
- Cohort-driven programs
- Building anticipation

**Pros**:
- Creates urgency
- Manageable onboarding bursts
- Marketing focused
- Premium positioning

**Cons**:
- Lost sales between windows
- Complex scheduling
- Must deliver on promises
- Launch fatigue potential

### Evergreen (Always Open)

**How It Works**:
- Anyone can join anytime
- Consistent marketing
- Automated onboarding
- Steady growth

**Best For**:
- Ongoing communities
- Lower price points
- Continuous content
- Scale-focused

**Pros**:
- Revenue flows constantly
- Simpler operations
- No launch stress
- Can automate fully

**Cons**:
- Less urgency
- Harder to create events
- Can feel less special
- Requires constant marketing

### Cohort Model

**How It Works**:
- Fixed start dates (monthly, quarterly)
- Members join in groups
- Progress together
- Graduate together

**Best For**:
- Transformation programs
- Course + community
- Accountability-focused
- B2B training

**Pros**:
- Strong peer bonding
- Built-in accountability
- Clear structure
- Measurable outcomes

**Cons**:
- Complex operations
- Limited enrollment windows
- Must fill each cohort
- Less flexibility

### Hybrid Approaches

**Open Community + Closed Programs**:
```
Evergreen community access
+ Quarterly intensive cohorts
+ Monthly challenges
+ Annual summits
```

**Rolling Cohorts**:
```
New cohort starts every month
Always accepting applications
Staggered member experiences
```

## Part 2: Pre-Launch Final Week

### The Week Before Launch

**Day -7: Final Content Review**
- Check all written content
- Test all videos/media
- Verify links work
- Review onboarding flow

**Day -6: Technical Testing**
- Payment processing test
- Email delivery test
- Mobile experience check
- Load test (if needed)

**Day -5: Team Briefing**
- Moderators know their role
- Support process clear
- Everyone knows timeline
- Emergency contacts set

**Day -4: Founding Member Activation**
- Remind founding members
- Brief them on launch support
- Ask for launch day engagement
- Prepare them for new members

**Day -3: Marketing Prep**
- All emails scheduled
- Social content ready
- Affiliates briefed (if any)
- Landing pages live

**Day -2: Soft Test**
- Invite a few more founding members
- Watch the complete flow
- Fix any issues
- Final adjustments

**Day -1: Rest and Prep**
- Get sleep
- Clear your schedule
- Prepare for high activity
- Send "tomorrow" teaser

## Part 4: Launch Week (Days 1-7)

### Day 1: Welcome Wave

**Your Focus**:
- Welcome EVERY new member
- Respond to ALL posts
- Fix any issues immediately
- Be highly visible

**What to Post**:
```
"Welcome to everyone who joined today! 🎉

We now have [X] members from [X] countries/industries/etc.

If you haven't yet:
1. Introduce yourself in #introductions
2. Check out [key resource]
3. Join us for [first event]

Questions? Drop them below!"
```

### Day 2: Orientation

**Your Focus**:
- Help members get oriented
- Point to key resources
- Start first discussions
- Address early questions

**What to Post**:
```
"Day 2 check-in!

How's everyone settling in?

If you need help:
• Finding something → Ask here
• Tech issues → DM me
• Not sure where to start → Check [Getting Started]

What's your biggest question right now? 👇"
```

### Day 3: First Value

**Your Focus**:
- Deliver early value
- Start content rhythm
- Encourage engagement
- Highlight early wins

**What to Post**:
```
"Time for our first [content/lesson/resource]!

[Brief description of value]

→ Check it out: [Link]

Who's diving in? What's your biggest takeaway?"
```

### Day 4: Connections

**Your Focus**:
- Facilitate member connections
- Start matching/networking
- Highlight common interests
- Encourage DMs

**What to Post**:
```
"I've noticed we have several members interested in [topic].

Tagging you all: @member1 @member2 @member3

You should definitely connect! Anyone up for a virtual coffee this week?"
```

### Day 5: First Event

**Your Focus**:
- Host first live event
- High energy and value
- Get members on camera/voice
- Create shared experience

**Event Structure**:
```
1. Welcome and energy (5 min)
2. Brief intro from host (5 min)
3. Member introductions (15-20 min)
4. Q&A or teaching (20 min)
5. Action items and close (10 min)
```

### Days 6-7: Rhythm Establishment

**Your Focus**:
- Continue consistent posting
- Establish weekly rhythms
- Gather early feedback
- Celebrate first wins

## Part 5: Onboarding Excellence

### The First 7 Days Experience

**Critical Window**: 70% of churn happens in first 30 days. The first week is crucial.

**Onboarding Goals**:
1. Member feels welcomed
2. Member knows how to navigate
3. Member takes first action
4. Member makes first connection
5. Member sees early value
6. Member forms habit

### Onboarding Sequence

**Hour 0-1 (Immediate)**:
```
✅ Welcome email received
✅ Account access confirmed
✅ Getting started guide visible
✅ First action clear
```

**Day 1**:
```
✅ Complete profile
✅ Read community guidelines
✅ Post introduction
✅ Receive welcome from founder/team
```

**Day 2-3**:
```
✅ Explore key areas
✅ Engage with other intros
✅ Start first content/lesson
✅ Receive check-in message
```

**Day 4-5**:
```
✅ Join first discussion
✅ Connect with a member
✅ Attend first event (if available)
✅ Ask first question
```

**Day 6-7**:
```
✅ Experience first win
✅ Provide feedback
✅ Establish engagement routine
✅ Know where to get help
```

### Activation Metrics

**Track These Actions**:
- Profile completed
- Introduction posted
- First comment made
- First resource accessed
- First event attended
- First DM sent

**Target**: 70%+ complete all by Day 7

## Part 7: Handling Launch Issues

### Common Launch Problems

**Problem: Payment Processing Fails**
```
Solution:
1. Have backup payment method ready
2. Communicate clearly to affected members
3. Fix immediately or offer manual processing
4. Follow up personally
```

**Problem: Platform Goes Down**
```
Solution:
1. Have status page ready
2. Communicate via email/social
3. Extend launch window if needed
4. Contact platform support urgently
```

**Problem: Low Signup Numbers**
```
Solution:
1. Don't panic (yet)
2. Extend launch window
3. Increase email frequency
4. Try different angles
5. Reach out personally
```

**Problem: Overwhelmed with Support**
```
Solution:
1. Create FAQ doc immediately
2. Batch similar questions
3. Post answers publicly
4. Recruit founding members to help
5. Prioritize urgent issues
```

**Problem: Founding Members Not Engaging**
```
Solution:
1. Personal outreach
2. Ask them to seed content
3. Create easy engagement opportunities
4. Offer incentives
5. Address any issues they have
```

## Part 8: Launch Marketing

### Email Marketing

**List Segmentation**:
```
Segment 1: Waitlist (never bought) → Full sequence
Segment 2: Past customers → Personalized offer
Segment 3: Engaged openers → Higher frequency
Segment 4: Cold subscribers → Fewer emails
```

**Email Frequency**:
```
Day -7: "Coming soon" teaser
Day -3: "Get ready"
Day -1: "Tomorrow"
Day 0: Launch email (morning)
Day 0: Update email (evening)
Day 1: Social proof
Day 2: FAQ/Objections
Day 3: Testimonials
Day 4: Scarcity reminder
Day 5: Last day warning
Day 6: Last hours
Day 7: Final call / Closed
```

### Social Media Strategy

**Content Mix**:
```
40% Direct promotion
30% Behind the scenes
20% Social proof/testimonials
10% Educational value
```

**Platform-Specific**:
- **Twitter/X**: Threads, real-time updates
- **LinkedIn**: Professional angle, longer posts
- **Instagram**: Stories, visual content
- **YouTube**: Launch announcement video

**If Using Affiliates**:
```

- Brief affiliates on offer

- Notify affiliates of launch
- Provide real-time stats

- Address issues quickly

- Thank affiliates
```

## Part 10: Launch Post-Mortem

### What to Review

**What Worked**:
- Which emails performed best?
- Which content drove signups?
- What did members love most?
- What exceeded expectations?

**What Didn't Work**:
- Where did we lose people?
- What caused friction?
- What complaints did we receive?
- What fell flat?

**What to Improve**:
- Onboarding refinements
- Content gaps to fill
- Process improvements
- Technical fixes

### Post-Launch Survey

```
POST-LAUNCH MEMBER SURVEY

Thank you for joining [Community]!

Quick feedback (2 minutes):

1. How satisfied are you so far? (1-10)

2. What's been most valuable?

3. What's been confusing or frustrating?

4. What would make this more valuable to you?

5. Would you recommend to a friend? (1-10)
   Why or why not?

6. Anything else we should know?
```

## Summary

### Launch Success Factors

1. **Pre-Work Matters**: Don't skip pre-launch
2. **Momentum Is Key**: Launch with activity, not silence
3. **Be Present**: Founder visibility crucial first week
4. **Fix Fast**: Address issues immediately
5. **Celebrate Wins**: Even small ones

### Launch Week Priorities

```
Day 1-2: Welcome and orient
Day 3-4: Deliver value and connections
Day 5-6: Events and engagement
Day 7: Recap and momentum
```

### Launch Success Metrics

**Green Light Targets**:
- 70%+ onboarding completion
- 50%+ Day 7 engagement
- <10% first-week churn
- Positive feedback majority
- Members helping members

**Warning Signs**:
- <50% complete onboarding
- Declining daily activity
- High support volume
- Negative sentiment
- Ghosted new members
